最近常聽到這些：

- 「我們在做 AI Agent 來自動審查 PR」
- 「我們在做 AI Agent 來自動寫測試」
- 「我們在做 AI Agent 來自動產生 release notes」
- 「我們在做 AI Agent 來自動回覆客服問題」

每次聽完，我都忍不住問：「那你怎麼告訴它要怎麼做？」

### 指令永遠得先寫

不管是 PR review、寫測試，還是回覆客服——在 AI 能動之前，有件事永遠要先完成：

**你得把「怎麼做」寫清楚。**

- PR review 要看什麼？命名規範？安全漏洞？效能問題？
- 測試要寫到什麼程度？unit test？integration test？
- release notes 的語氣和格式是什麼？
- 客服回覆的邊界在哪？哪些問題不能回？

這些不寫下來，AI 只是在亂猜。寫下來之後，所謂的「Agent」才有辦法動。

### 那 Agent 加了什麼？

寫好指令之後，Agent 加的是**自動觸發**：

```
事件發生 → 帶著指令呼叫 AI → 把結果送到對的地方
```

以 PR review 為例：

```
PR 開啟 → webhook 觸發 → AI 執行審查（依照你寫的規則）→ 自動留言
```

這部分是真正的工程工作：串接 webhook、呼叫 API、處理錯誤。不是沒有價值，但它不是「AI」，它是水管（plumbing）。

### 所以「做一個 Agent」的工作量在哪？

| 工作 | 難度 | 這是 AI 嗎？ |
|---|---|---|
| 把規則和期望寫清楚 | 需要領域知識，不容易 | 不是，這是文件 |
| 串接觸發器與 API | 工程工作 | 不是，這是水管 |
| 讓 AI 執行 | 呼叫 API | 這才是 |

真正難的往往是第一項：**你得先想清楚自己要什麼**。這件事 AI 幫不了你。

### 結論

下次聽到「我們在做 AI Agent 來做 X」，可以直接問：

> 「你們的指令寫好了嗎？」

如果還沒，那 Agent 根本還沒開始。不管架構多漂亮，觸發器接得多完整，沒有好的指令，AI 只是一個很貴的亂猜機器。
